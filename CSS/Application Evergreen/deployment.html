Deployment Process

Setup Evergreen Component

Application evergreen component is setup as a batch process in a scheduled manner. For every run, it creates a PR to the respective repo in stash. If no changes, JIRA will be updated with comments.

Identify Vulnerabilities

Developer/Lead SME gets Prisma/Veracode/any analysis tool report as an input and identifies the vulnerabilities to be fixed.

Validate PR

Developer validates the PR generated. If the PR is not recently created, the developer manually triggers the Application evergreen component and gets the new changes. If satisfied with the dev testing, proceed.

Review and Approve PR

Development team members review and approve the PR.

Create Release Cut

Post-approval, the developer creates a release cut from the develop branch.

Deploy to UAT & Prod

The release tag is deployed to lower and prod regions via Jenkins/DTP pipeline.

Notify Testing Team

Notify the testing team about the UAT deployment. If no issues are encountered, plan the production release.

Monitor Logs

Monitor logs for a week to ensure no issues arise.

Manual Fix Details

The legacy dependency 'namea' requires manual intervention to update due to its complexity. The fix will be implemented in the feature branch 'fix/legacy-dependency-namea' and will be deployed in the next scheduled release.
